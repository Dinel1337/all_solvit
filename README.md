# Solvit — портфолио бэкенд-проектов (3/8)

**Идея:** разработать 8 полноценных пет-проектов, которые покрывают основные сценарии реальной разработки: трекинг тренировок, квиз-платформа, телеграм-боты, парсинг, финансы, мониторинг, аналитика.

Каждый проект построен на **FastAPI** с использованием общей библиотеки `_core` по принципу DDD (shared kernel), что обеспечивает единообразие архитектуры, аутентификации, работы с БД и формата ответов.

## Быстрый старт

### Автоматическая установка (рекомендуется)
```bash
curl -sSL https://raw.githubusercontent.com/Dinel1337/all_solvit/main/bin/setup.sh | bash
```
**Скрипт автоматически:**

| | Действие |
|---|----------|
| ✅ | Проверит наличие Docker, Docker Compose, uv, Git |
| 📦 | Склонирует репозиторий (если нужно) |

## Запуск проектов

### Автоматический (контейнеризированный) — рекомендую

```bash
docker-compose up -d
```
### Терминальный (точечный запуск с подробными логами)

```bash
uv run python main.py -t -n{номер проекта}
```

| Проект | Команда | Порт | Документация |
|--------|---------|------|--------------|
| API Tracker | python main.py -t -n 3 | 8000 | http://localhost:8000/docs |
| Quiz Platform | python main.py -t -n 2 | 8001 | http://localhost:8001/docs |
| Chat Bot Binance | python main.py -t -n 1 | — | Telegram-бот(нужен впн или настройка проси в aiogram3) |

**Дополнительные флаги:**

| Флаг | Описание |
|------|----------|
| `-t, --terminal` | Локальный запуск (обязательно для терминального режима) |
| `--init-db` | Только инициализация БД (создание схем) |
| `-i, --info` | Показать список доступных проектов |

## Подход к документированию

Каждый проект документируется на четырёх уровнях:

### 1. Документация в `__init__.py` (вложенные модули)

- **Нижний уровень** (папки типа `auth/`, `database/`, `exceptions/`, `models/`, `repository/`, `schemas/`, `service/`) — каждый `__init__.py` содержит docstring, описывающий **конкретный модуль**: его назначение, экспортируемые классы/функции, примеры.

- **Средний уровень** (папки типа `api/`, `local_core/`) — `__init__.py` собирает документацию из дочерних модулей и описывает **общее покрытие** раздела (например, «API роутеры для упражнений, категорий, групп мышц»).

### 2. Корневой README проекта

В корне каждого сервиса (`api_tracker/`, `quiz_platform/`, `chat_bot_binance/`) лежит **развёрнутый README.md**, который включает:
- функциональность и бизнес-логику
- список всех эндпоинтов с таблицей
- схемы данных
- технические детали (зависимости от `_core`, JWT, настройки)
- примеры запросов/ответов
- скриншоты Swagger UI или анимацию работы

### 3. Демонстрация (скриншоты / анимация)

Визуальные материалы лежат в `docs/screenshots/` каждого проекта и вставляются в README:
- Swagger UI — все эндпоинты, схемы, возможность тестирования
- анимация работы бота (для `chat_bot_binance`)
- схема архитектуры «как `_core` используется проектами»


Таким образом, документация идёт от частного (внутри модуля) к общему (портфолио), что упрощает навигацию и поддержку.
